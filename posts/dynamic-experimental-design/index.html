Experimental Woes

    14 Apr 2021

    Dynamic experimental design in 100 lines of Julia code

    Optimal experimental design is an area of statistics focused on constructing informative experiments. In this tutorial we show how to construct such informative experiments for dynamic systems using only 100 lines of Julia code. We will work with a well-mixed fed-batch bioreactor as an example system. We have quite a bit of domain knowledge how to model the behavior of a bio-reactor. The reactor has three dynamic states: the substrate concentration $C_s$, the biomass concentration $C_x$ and the volume of the reactor $V$. The evolution in time of these states is governed by the following differential equations:

    \[ \frac{dC_s}{dt} = -\sigma C_x + \frac{Q_{in}(t)}{V}(C_{S,in} - C_s) \]

    \[ \frac{dC_x}{dt} = \mu C_x - \frac{Q_{in}(t)}{V}C_x \]

    \[ \frac{dV}{dt} = Q_{in}(t) \]

    where,

    \[ \mu = \frac{\mu_{max}C_s}{K_s + C_s} \]

    \[ \sigma = \frac{\mu}{y_{x,s}} + m \]

    Not all parameters in these equations are exactly known. We are unsure of the value of the maximal growth rate $\mu_{max}$, and the half saturation constant $K_s$. We want to construct an experiment to learn about these parameters. To reach this goal we can control the volume of the reactor by the volumetric flow rate $Q_{in}(t)$. And we will measure the two states $C_s$ and $C_x$. (The other parameters in the equation, namely the substrate feed concentration $C_{S,in}$, the maintenance factor $m$ and yield $y_{x,s}$, are exactly known)

    Most of the numerical values are taken from Telen et al. (2014).

    Defining the system.

    First we define the dynamics of the fed-batch reactor.

    function dynamics!(du,u,p,t)
    Cₛ, Cₓ, V = u   
    μₘₐₓ, Kₛ = @view p[1:n_θ]
    control_par = @view p[n_θ+1:end]
    Qᵢₙ = control_network(t, control_par)[1]
    Cₛ_ᵢₙ, yₓ_ₛ, m = 50.0, 0.777, 0.0
    μ = μₘₐₓ*Cₛ/(Kₛ + Cₛ)
    σ = μ/yₓ_ₛ + m

    du[1] = -σ*Cₓ + Qᵢₙ/V*(Cₛ_ᵢₙ - Cₛ)
    du[2] = μ*Cₓ - Qᵢₙ/V*Cₓ
    du[3] = Qᵢₙ
    return nothing
end

    Finding the optimal controls is an infinite dimensional optimization problem. To reduce the complexity of this problem to a non-linear optimization one, we parametrize the control function. We call this parametrization x, and in this case we use a neural network. Note that this neural network constrains the control input between 0 and 1.

    control_network = FastChain(FastDense(1, 5, tanh),FastDense(5, 1,sigmoid))

    Some optimal experimental design theory

    Generally we do not measure all the states continuously, but instead we take measurements $y_i$, which are some function of the states at discrete time points.

    \[ \frac{du}{dt} = f(u,\theta,x,t) \]

    \[ y_k = h(u(t_k)) \]

    In our case we measure the substrate and biomass concentration every hour. These measurements should inform us about the true value of the uncertain parameters. Often these measurements are noisy, with covariance $\Sigma$

    tmeasurement = 0.0:1.0:15.0
Σ = [0.001 0.0; 0.0 0.000625]

    The Fisher information matrix $I$ is a popular way to quantify this information content of an experiment,

    \[ I(\theta,x) = \sum_k \frac{\partial y_k'(\theta,x)}{\partial\theta}\Sigma^{-1} \frac{\partial y_k(\theta, x) }{\partial\theta} \]

    The intuition behind this formula is that in a good experiment the measurements should be sensitive towards the value of the uncertain parameters. If the experiment is such that the measurements are similar no matter what the value of the true parameter is, then it will be hard to precisely determine that parameter value. These sensitivities of the measurements towards the uncertain parameters can be further expanded,

    \[ \frac{\partial y_k(\theta,x)}{\partial\theta} = \frac{\partial y_k(\theta,x)}{\partial u} \frac{\partial u(\theta,x)}{\partial\theta}. \]

    Since we only measure the first factor is equal to [1 0; 0 1; 0 0]. The second factor can be calculated from the forward sensitivity transform of the differential equation system

    \[ \frac{d}{dt}\frac{\partial u(\theta,x)}{\partial\theta} = \frac{\partial}{\partial\theta}\frac{du(\theta,x)}{dt} = \frac{\partial f(u,\theta,x)}{\partial u}\frac{\partial u}{\partial \theta} + \frac{\partial f(u,\theta,x)}{\partial \theta} \]

    \[ \frac{\partial u(t=0)}{\partial \theta} = 0 \]

    We can implement this in Julia as

    function dynamics_forward_sensitivity!(duₑₓₜ,uₑₓₜ,p,t)
    du = @view duₑₓₜ[1:n_u]
    u = @view uₑₓₜ[1:n_u]
    dynamics!(du,u,p,t)
    du_sen_mat = reshape(@view(duₑₓₜ[n_u+1:end]),n_u,n_θ)
    u_sen_mat = reshape(@view(uₑₓₜ[n_u+1:end]),n_u,n_θ)
    θ = @view p[1:n_θ]
    x = @view p[n_θ+1:end]
    ForwardDiff.jacobian!(du_sen_mat, (du,θ)->dynamics!(du,u,vcat(θ,x),t), du, θ)
    Ju = ForwardDiff.jacobian( (du,u)->dynamics!(du,u,p,t), du, u)
    mul!(du_sen_mat,Ju,u_sen_mat,1.0,1.0)
    return nothing
end
u₀ₑₓₜ = vcat(u₀, zeros(n_u*n_θ))
duₑₓₜ = similar(u₀ₑₓₜ)
probₑₓₜ = ODEProblem(dynamics_forward_sensitivity!,u₀ₑₓₜ,tspan,p) 
solₑₓₜ = solve(probₑₓₜ ,Tsit5(),saveat=tmeasurement,reltol=1e-5)

    We want the Fisher information matrix to be as large as possible, but what constitutes a large matrix? The inverse of the FIM is related to confidence ellipsoids and we want these ellipsoids to be as small as possible, which turns out to be equivalent to maximizing the determinant of the FIM. In experimental design literature these are called D-optimal experiments. In Julia code this can be done in the following way,

    function D_criterion(solₑₓₜ::ODESolution)
    FIM = zeros(eltype(solₑₓₜ),n_θ,n_θ)
    for k in 1:length(solₑₓₜ)
        du_dθ = zeros(eltype(solₑₓₜ),n_u,0)
        u = solₑₓₜ.u[k][1:n_u]
        for i = 1:n_θ
            du_dθ= hcat(du_dθ,solₑₓₜ.u[k][n_u*i+1:n_u*(i+1)])
        end
        dy_θ = du_dθ[1:2,:]
        FIM += dy_θ'*inv(Σ)*dy_θ
    end
    return det(FIM)
end

    The major issue with experimental design based on the Fisher information matrix, is the dependence of the optimal design on the true parameter value that the experiment aims to learn about. We can robustify this by averaging the D-criterion over a distribution of possible parameter values. This method requires that we specify the uncertain parameters as probability distributions provided by Distributions.jl. We use truncated normal distributions centered around the initial guesses we used earlier. 

    function robust_D_criterion_manual(x)
    Ed  = zero(eltype(x))
    N = 1_000
    Random.seed!(123)
    for i = 1:N
        θ_dist  = [truncated(Normal(0.421,0.1),0.3,0.6),truncated(Normal(0.439,0.1),0.3,0.6)]
        p_test = vcat(rand.(θ_dist),x)
        prob_test = ODEProblem(dynamics_forward_sensitivity!,u₀ₑₓₜ,tspan,p_test)  
        sol_test = solve(prob_test, Tsit5();
                                            saveat=tmeasurement,
                                            sensealg=QuadratureAdjoint(reltol=1e-5,autodiff=true,autojacvec=false),
                                            reltol=1e-5)
        Ed += D_criterion(sol_test)
    end
    Ed/N
end

    Finally, we get to optimizing the controls. A first order optimization technique is used. Optim.jl is capable of calculating the required gradient using automatic differentiation. (switch this to reverse mode...) 

    xₒₚₜ = optimize(x->-robust_D_criterion_manual(x), xᵢₙᵢ, ConjugateGradient(); autodiff = :forward).minimizer

    Now all that remains is showing the added value of the optimal experiment. We simulate many experiments according to both the design we started with (control parameters put to zero), and the optimized design, using the true parameter values. We then see how precisely we can recover the true parameters from the simulated data sets.

    function SSE(θ,x)
    prob = ODEProblem(dynamics!,u₀,tspan,vcat(θ,x)) 
    sol = solve(prob,Tsit5(),saveat=tmeasurement,reltol=1e-5)
    y_est = sol[1:2,:]
    sum((y_measure - y_est).^2)
end
y_true  = solve(ODEProblem(dynamics!,u₀,tspan,vcat(θₜ,xᵢₙᵢ)),Tsit5(),saveat=tmeasurement,reltol=1e-5)[1:2,:]
θ_estimates = zeros(n_θ,100)
for j = 1:100
    global y_measure = y_true + rand(MvNormal(Σ),length(tmeasurement))
    θ_estimates[:,j] = optimize(θ->SSE(θ,xᵢₙᵢ),  θₜ).minimizer
end
Plots.scatter(θ_estimates[1,:],θ_estimates[2,:],label="unoptimized experiment")
